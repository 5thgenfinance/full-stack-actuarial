# Full Stack Actuarial - User Guides Catalog

Welcome to the Full Stack Actuarial documentation repository. This README serves as a central catalog of all available user guides to help you navigate the documentation and find the resources you need.

## Documentation Structure

The user guides are organized by topic and technology. Each guide provides comprehensive instructions and best practices for its respective area.

---

## User Guides

### 1. JSON User Guide
**File:** `json-user-guide.md`

**Purpose:** A comprehensive guide to working with JSON (JavaScript Object Notation) format

**Content Includes:**
- JSON syntax and structure fundamentals
- Data types and formatting rules
- Best practices for JSON file creation and validation
- Integration with actuarial systems and data workflows
- Common JSON use cases in the actuarial full stack

**Best For:** Developers, data analysts, and technical professionals working with JSON data structures

**Key Topics:**
- JSON syntax and validation
- Working with nested objects and arrays
- JSON schema design
- Error handling and troubleshooting

---

### 2. Markdown User Guide
**File:** `markdown-guide.md`

**Purpose:** A complete guide to writing and formatting documentation using Markdown

**Content Includes:**
- Markdown syntax and formatting options
- Creating headers, lists, tables, and code blocks
- Structuring technical documentation
- Best practices for readability and organization
- Integration with GitHub and documentation platforms

**Best For:** Technical writers, developers, and documentation specialists

**Key Topics:**
- Basic Markdown syntax
- Advanced formatting techniques
- Creating well-structured documents
- Markdown in version control systems

---

### 3. QA Environment User Guide
**File:** `qa-env-user-guide.md`

**Purpose:** A comprehensive guide for setting up, configuring, and working in the Quality Assurance environment

**Content Includes:**
- QA environment setup and configuration
- Testing strategies and workflows
- Test case development and execution
- Defect tracking and reporting
- Integration with CI/CD pipelines
- Best practices for quality assurance processes

**Best For:** QA professionals, testers, and quality assurance team members

**Key Topics:**
- Environment configuration
- Testing procedures and protocols
- Defect management
- Performance and regression testing

---

### 4. YAML User Guide
**File:** `yaml-user-guide.md`

**Purpose:** A detailed guide to YAML (YAML Ain't Markup Language) format and its applications

**Content Includes:**
- YAML syntax and structure fundamentals
- Data types, lists, and nested objects
- Configuration file best practices
- YAML in CI/CD and deployment pipelines
- Validation and error handling
- Integration with actuarial and IT systems

**Best For:** DevOps engineers, system administrators, developers, and infrastructure teams

**Key Topics:**
- YAML syntax and data structures
- Configuration file management
- Deployment and automation workflows
- Error prevention and validation
